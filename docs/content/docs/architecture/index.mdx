OptimusKG is built on top of the [Kedro framework](https://kedro.org/) and follows a **medallion architecture** pattern to transform raw biomedical data into a unified knowledge graph.

## Core Components

The pipeline is composed of the following components:

| Component | Description |
| --- | --- |
| [Layers](/docs/architecture/medallion-layers) | Medallion architecture: landing, bronze, silver, gold |
| [Catalog](/docs/architecture/catalog) | Dataset definitions with schemas, checksums, and metadata |
| [Pipelines](/docs/architecture/pipelines) | DAG-based workflows composed of pure Python functions |
| [Hooks](/docs/architecture/hooks) | Lifecycle callbacks for validation and data downloads |
| [Runners](/docs/architecture/runners) | Execution engines for parallel and dry-run modes |
| [Datasets](/docs/architecture/datasets) | Custom dataset types for OWL, XML, SQL dumps, etc. |
| [Providers](/docs/architecture/providers) | Automatic data download from external sources |
| [Configuration](/docs/architecture/configuration) | Environment-based settings management |

## Data Flow

```
External Sources -> Landing -> Bronze -> Silver -> Gold -> Export (Parquet/Neo4j)
```

1. **Landing**: Raw data files downloaded from external sources via [Providers](/docs/architecture/providers)
2. **Bronze**: Extracted and standardized Polars DataFrames
3. **Silver**: Consolidated entities and relationships across sources
4. **Gold**: Final knowledge graph exported via BioCypher

## Technology Stack

- **Kedro** - Pipeline orchestration and project structure
- **Polars** - All data transformations (not Pandas)
- **BioCypher** - Knowledge graph schema and export
- **Docker** - Neo4j database for graph exploration
