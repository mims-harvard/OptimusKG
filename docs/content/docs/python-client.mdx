The [`optimuskg`](https://pypi.org/project/optimuskg/) PyPI package is a thin client for loading the published knowledge graph. It resolves Dataverse file IDs automatically, caches downloads locally, and returns Polars DataFrames or a NetworkX graph.

Requires Python 3.12 or newer. The package pulls in `polars`, `pyarrow`, `networkx`, `requests`, and `platformdirs`.

## Quick Start

```python
import optimuskg

nodes, edges = optimuskg.load_graph(lcc=True)
G = optimuskg.load_networkx(lcc=True)
```

The first call downloads `lcc_nodes.parquet` and `lcc_edges.parquet` from Harvard Dataverse. Subsequent calls return the cached copies.

## API Reference

### `get_file(relative_path, *, force=False) -> Path`

Download (if missing) and return the local cache path for a single parquet file. Paths mirror the layout under `data/gold/kg/parquet/` in the source repository:

```python
optimuskg.get_file("nodes.parquet")                 # full nodes table
optimuskg.get_file("edges.parquet")                 # full edges table
optimuskg.get_file("lcc_nodes.parquet")             # LCC nodes table
optimuskg.get_file("lcc_edges.parquet")             # LCC edges table
optimuskg.get_file("nodes/gene.parquet")            # only Gene nodes
optimuskg.get_file("edges/disease_gene.parquet")    # only DIS-GEN edges
```

Pass `force=True` to re-download even if the file is already cached.

### `load_parquet(relative_path, *, force=False, **kwargs) -> pl.DataFrame`

Thin wrapper around `get_file` + `polars.read_parquet`. Extra keyword arguments are forwarded to `pl.read_parquet`.

```python
drugs = optimuskg.load_parquet("nodes/drug.parquet")
only_ids = optimuskg.load_parquet("nodes/drug.parquet", columns=["id"])
```

### `load_graph(*, lcc=False, force=False) -> tuple[pl.DataFrame, pl.DataFrame]`

Return the full graph (or its largest connected component) as two Polars DataFrames: `(nodes, edges)`.

```python
nodes, edges = optimuskg.load_graph()          # full graph
nodes, edges = optimuskg.load_graph(lcc=True)  # LCC only
```

Column schemas match the raw parquet files — see [Node Types](/docs/graph-schema/node-types) and [Edge Types](/docs/graph-schema/edge-types).

### `load_networkx(*, lcc=False, force=False, parse_properties=True) -> nx.MultiDiGraph`

Build a `networkx.MultiDiGraph` from the nodes and edges DataFrames.

- Each row's `properties` JSON string is decoded into a dict and splatted onto the node/edge attributes (set `parse_properties=False` to keep the raw string under a single `properties` attr).
- Every edge carries an `undirected` boolean indicating whether the source-of-truth edge is undirected. The graph type is always `MultiDiGraph` so that direction is preserved; flip to an undirected view with `G.to_undirected()` if you need one.
- Calling this without `lcc=True` emits a warning — the full graph (21M edges) requires several GB of memory.

```python
G = optimuskg.load_networkx(lcc=True)
print(G.number_of_nodes(), G.number_of_edges())

# Filter by node label
genes = [n for n, attrs in G.nodes(data=True) if attrs["label"] == "GEN"]

# Filter by relation type
expression_edges = [
    (u, v) for u, v, attrs in G.edges(data=True)
    if attrs["relation"] == "EXPRESSION_PRESENT"
]
```

## Configuration

### Cache directory

Downloaded files are stored under `platformdirs.user_cache_dir("optimuskg")` by default — that is `~/Library/Caches/optimuskg` on macOS and `~/.cache/optimuskg` on Linux. Files are keyed by `<doi_slug>/<dataset_version>/<relative_path>`, so a new Dataverse release invalidates the cache automatically.

Override the location from code:

```python
optimuskg.set_cache_dir("/data/optimuskg-cache")
```

Or from the environment, before your script starts:

```bash
export OPTIMUSKG_CACHE_DIR=/data/optimuskg-cache
```

### Pointing at a different dataset

For pre-release or private datasets you can change the DOI:

```python
optimuskg.set_doi("doi:10.7910/DVN/EXAMPLE")
```

Or via `OPTIMUSKG_DOI` in the environment. Use `optimuskg.set_server(...)` / `OPTIMUSKG_SERVER` for a non-Harvard Dataverse installation.
