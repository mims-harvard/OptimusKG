OptimusKG integrates data from external biomedical sources and ontologies. Direct sources are ingested into the pipeline, while indirect sources are incorporated through provenance within the corresponding direct resources. Each source is annotated with its enum value (in parentheses).

## Ontologies

Ontologies provide controlled vocabularies, hierarchical organization, and cross-references across biomedical entities.

| Source | Description | Version |
| --- | --- | :-: |
| Gene Ontology (GO+) | Functional annotation ontology for biological processes, cellular components, and molecular functions | current |
| Mondo | Unified disease ontology integrating multiple disease classification systems | current |
| Human Phenotype Ontology | Standardized vocabulary for describing human phenotypic abnormalities | v2025-05-06 |
| Uberon | Cross-species anatomy ontology for tissues and organs | v2025-05-28 |
| Disease Ontology | Structured disease classification used for alignment and mapping across resources | BioOntology v656 |
| Orphanet | Rare disease ontology with clinical and epidemiological annotations | BioOntology v30 |
| Biolink Model | High-level schema defining entity types and relationships in biomedical knowledge graphs | v3.2.1 |
